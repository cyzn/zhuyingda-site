zippo-ui

开发这个组件库的初衷主要是：

  • 将前端常用UI组件的交互逻辑抽象出来进行封装，提高开发效率
  • 将ie环境下的bug在组件层面解决掉，避免重复的修bug
  • 支持npm包管理方式在commonJs环境下开发，支持webpack、browserify等打包工具

主要特点：

  • 轻量级：没有引入任何其他类库，代码体积小
  • 易使用：api接口尽可能简单，降低学习成本和维护成本
  • 基于jquery，主要解决国内PC端页面，向ie6兼容
  • 提供umd版本，并且拥有整合版以及每一个子功能模块分别对应的打包js
  • 支持在commonJs的前端代码预编译环境，如webpack、browserify
  • 同时逐渐支持H5,ES6等新规范，同样为移动端页面提供相应优化

安装方式

1.外链js的引入方式

将下列代码粘贴到你的网页中，放在jquery之后： <script src="http://www.zhuyingda.com/static/js/zippo.release.js"></script>

2.commonJs环境下引入方式

首先安装zippo-ui： npm install --save-dev zippo-ui 然后在你的代码中引入zippo-ui，并按照下面的api文档调用各组件的api即可。建议以如下方式引入，以跟api上面的命名保持一致： var zippo = require("zippo-ui");

弹层组件(mask)：

显示弹层：

var m = zippo.mask.showMask("<div>这里是弹层内容</div>",{animation:'fade'});

关闭弹层：

m.close();

抖动弹层：

m.shake();

分页组件(pager)：

分页组件初始化：

zippo.pager.init({
    $el: $('#pager'),
    btnInPage: 10,
    itemInPage: 14,
    onTurn: function (p) {
        console.log(p);
    }
});

设置数据条数：

zippo.pager.setTotal(70);

轮播图(slider)：

组件初始化：

var c = zippo.carousel.init({
    $el: $('#carousel'),
    width: 200,
    itemWidth: 100,
    height: 100,
     res:
    [
        '<p>123</p>',
         '<p>123</p>',
        '<p>123</p>',
        '<p>123</p>'
    ]
});

向左滑动：

c.turnLeft();

向右滑动：

c.turnRight();
